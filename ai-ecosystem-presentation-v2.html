Layer 1

🧠AI — Artificial Intelligence

AI is the broadest term here. It means any system that mimics or simulates human intelligence.

  • Recognizing images (computer vision)
  • Playing chess or Go (game AI)
  • Recommending videos (recommendation engines)
  • Understanding and generating language (LLMs)
Analogy
Think of AI like "Vehicles." It's the category. Cars, trucks, motorcycles — they're all vehicles, but they work differently. LLMs are one type of AI vehicle.

Not all AI is LLMs. Spam filters and self-driving cars are AI too.

Layer 2

📖LLM — Large Language Model

An LLM is a specific type of AI — one trained to predict the next word in a sequence. Train it on enough data, and it starts to seem like it understands.

Tokens

The smallest unit of text an LLM processes. Not exactly words — can be a word, part of a word, or punctuation.

1000 tokens ≈ 750 words

Context Window

The maximum text the LLM can "hold in memory" at once — your prompt + its response combined.

Like working memory

Temperature

Controls randomness. Low (0.1) = predictable. High (0.9) = creative.

Textbook vs. Creative dial

Layer 3

🔌API — Application Programming Interface

An API is how programs talk to each other. Instead of you typing into a chat window, your code sends a message to the LLM and gets a response back.

// Your code sends a request to the LLM POST https://api.openai.com/v1/chat/completions Body: { "model": "gpt-4", "messages": [ { "role": "user", "content": "Hello!" } ] } // Response comes back { "choices": [{ "message": { "content": "Hi there!" } }] }
Analogy
An API is like a phone call. You dial a number (the endpoint), say something (the request), and get a reply (the response). The LLM is on the other end of the line.

Layer 4b — Skills

🎓Agent Skills

Skills are pre-built instruction sets that teach an agent how to behave in specific scenarios. They define the agent's personality, workflow style, and rules of engagement.

Analogy
If tools are what an agent can touch, skills are what an agent knows. Like the difference between having a wrench (tool) and knowing plumbing (skill).

📋 Custom Instructions

Define how the agent should respond — tone, format, level of detail, coding style preferences.

🎯 Task-Specific Skills

"Always write tests first", "Explain before coding", "Ask before deleting files" — behavioral rules.

📁 .clinerules / AGENTS.md

Project-specific skill files that live in your repo. The agent reads them automatically on every session.

🔄 Skill Modes

Switch between "architect mode" (plan first) and "builder mode" (code immediately) depending on the task.

# Example: .clinerules file Always explain your reasoning before writing code. Write tests before implementation (TDD). Never remove existing functionality without asking. Use TypeScript for all new files. Keep functions under 50 lines.

Essential Tool

📦Git — Version Control

Git is the foundation of modern software development. It tracks every change to your code, lets you experiment safely, and enables collaboration.

🕐 Time Machine

Every saved state is recoverable. Made a mistake? Go back to any previous version instantly.

🧪 Safe Experimentation

Branches let you try new ideas without breaking working code. Merge when ready, discard if not.

👥 Collaboration

Multiple people work on the same project simultaneously. Git merges changes and flags conflicts.

🤖 AI Agent Safety Net

When AI agents edit your code, Git lets you review, accept, or revert every change.

Analogy
Git is like Ctrl+Z on steroids. Instead of just undoing your last action, it saves every version of your project forever.
# The commands you'll use 90% of the time git init # Start tracking a project git add . # Stage all changes git commit -m "msg" # Save a checkpoint git branch feat # Create a parallel workspace git push # Upload to GitHub/GitLab git log --oneline # See history

Layer 5

🔗MCP — Model Context Protocol

MCP is a standard protocol that lets AI agents connect to external tools and data sources — file systems, databases, web browsers, and more.

Analogy
Before MCP: Every phone had its own charger. Every tool needed custom code.
After MCP: USB-C. One standard port. Plug in anything — the agent knows how to talk to it.

📁 File Systems

Read, write, search files on your machine

🗄️ Databases

Query data from SQL, NoSQL, etc.

🌐 Web Browsers

Scrape pages, automate web tasks

📦 Git Repos

Commit, branch, review, and push code

Context Management

🎯Context Engineering

What you put in determines what you get out — and it costs more than you think.

"Context engineering is the delicate art and science of filling the context window with just the right information for the next step." — Andrej Karpathy

Problem 1: Context is expensive

  • 10K tokens = ~$0.03
  • 200K tokens = ~$0.60
  • 600K tokens = $1.50+

More context ≠ better results. You're paying for noise.

Problem 2: Context makes models dumber

Smart Zone (0-50%) Dumb Zone (>50%)

The MCP Trap: Loading every server starts you in the dumb zone.

The Benchmark: Stay under 50% utilization when possible.

Problem 3: Bad Context Poisons Everything
Once bad information (task mixing, massive logs, hallucinations) enters the context window, the model treats it as ground truth. The fix: Start a new session. Don't try to fix it — reset it.

Context Management

🧠Four Ways to Manage Context

Every context engineering decision falls into one of these categories.

✍️ Write Context

Persist information outside the window so it's available without filling it up.

Scratchpads, memory files, CLAUDE.md

🎯 Select Context

Pull only what's relevant for this step — not everything that might be useful.

RAG, @-mentions, file references

🗜️ Compress Context

Reduce what's already in the window to just the tokens the task actually needs.

Summarization, trimming logs

✂️ Isolate Context

Split work across separate sessions so no single window accumulates everything.

Parallel agents, task separation

The Workflow Deep Dive

🔍Executing the RPI Framework

1. Research (Ask Mode)

  • Understand: How does this actually work?
  • Find: Where does this code live?
  • Stay objective: No implementation yet
Output: Research Document (files, flow, edge cases)

2. Plan (Architect Mode)

  • Outline: File & function names
  • Include testing: How to verify
  • Be explicit: What changes, what won't
Output: Plan File (step-by-step instructions)

3. Implement (Code Mode)

  • Execute: Step by step, no improvising
  • Keep context low: Fresh task per step
  • Commit frequently: Reversible changes
Key Insight: Hard thinking is already done!

Human review at research and planning stages is the highest leverage use of your time.

Best Practices

🌱The Habit That Ties It Together

You don't need to think about all strategies on every task. You need one default behavior.

1. One task per session

Finish the task, close the session. Start fresh for the next one.

2. Watch the meter

If you're past 50%, you're already in the dumb zone. Wrap up or restart.

3. When in doubt, restart

Something feels off? Don't debug the AI. Start a new session.

Frequent new sessions aren't a sign of confusion — they're a sign of discipline.

Tool Mastery

⚙️Configuring Your Agent

Mental Model for Configuration

  • AGENTS.md: Always-on rules ("README for AI agents"). Project conventions, build commands.
  • SKILLS.md: Task-specific workflows (on-demand). Reusable playbooks.
  • Modes: Personas with different behaviors (Architect, Coder).

Note: Memory Bank is being deprecated in favor of AGENTS.md + SKILLS.md

Working with Platform APIs

  • Swagger/OpenAPI: Pull directly via @file (Best for Standard REST)
  • Markdown: Document in AGENTS.md (Best for stable internal APIs)
  • URL/URI: Point to hosted docs (Best for rapidly changing APIs)
  • MCP Server: Build a custom connector (Best for multi-step interactions)
Top Settings
Start with defaults for Agent Behavior, Auto-Approve, and Autocomplete. Tune as you learn what works for your workflow.

The Future

🔮What to Keep in Mind Going Forward

The AI ecosystem moves fast. Here is how to stay ahead.

🛠️ Track New Dev Tools

Keep up to date with new tools for software development. The landscape changes frequently with new CLI agents, IDE extensions, and orchestrators.

✨ Updated Models

Watch for new frontier models. Capabilities jump significantly with each generation (e.g., Claude 3.5 to 3.7, GPT-4 to GPT-4.5).

🔑 Access & API Keys

Constantly research how to get free access, credits, or optimized API routing (like OpenRouter or GCP Free Tier) to minimize token costs.

🛡️ Security & Best Practices

Always stay informed about security updates. Never paste secrets, review agent code, and understand the data retention policies of your API providers.
