Introduction

What is GCP?

Google Cloud Platform is Google's cloud computing service. It gives you access to infrastructure you can rent by the second.

💻 Compute

Virtual machines, serverless functions, containers

💾 Storage

File storage, databases, object storage

🤖 AI/ML

Pre-built AI APIs, Vertex AI, GPU instances

🌐 Networking

Load balancers, DNS, CDN

Analogy
If your laptop is a single apartment, GCP is an entire city of infrastructure you can rent by the second.

Free Credit

The $300 Free Credit

  • Valid for 90 days from signup

Core Concept

What is a VM Instance?

A Virtual Machine is a computer running inside Google's data centers. You pick the CPU, RAM, storage, and OS — and it works exactly like a real computer.

Analogy
It's like renting a computer in Google's building. You SSH into it, install software, run code — but it lives in their data center, not on your desk.

Getting Started

Creating Your First VM

  1. Go to console.cloud.google.com
  2. Navigate to Compute Engine → VM Instances
  3. Click Create Instance
  4. Configure:
    • Name: my-first-vm
    • Region: us-central1 (cheapest, free-tier eligible)
    • Machine: e2-micro (free) or e2-medium (more power)
    • Boot disk: Ubuntu 22.04 LTS
    • Firewall: Allow HTTP and HTTPS
  5. Click Create — wait ~60 seconds

Hands-On

Connect & Explore

# SSH into your VM from terminal gcloud compute ssh my-first-vm --zone=us-central1-a # Or use browser-based SSH in GCP Console # Once connected, explore: uname -a # Check OS free -h # Check RAM df -h # Check disk space nproc # Check CPU cores # Install tools: sudo apt update sudo apt install python3 python3-pip git nodejs npm
Tip
Browser-based SSH requires no setup — just click "SSH" next to your VM in the console.

Important

Cost Management

Do This

  • Set budget alerts at 25%, 50%, 75%, 90%
  • Stop VMs when not in use
  • Use us-central1 (cheapest region)
  • Delete unused disks

Don't Do This

  • Leave GPU instances running
  • Use expensive regions (europe, asia)
  • Forget about disk costs
  • Upgrade to paid accidentally
# Stop a VM (stops compute billing) gcloud compute instances stop my-first-vm --zone=us-central1-a # Start it again when needed gcloud compute instances start my-first-vm --zone=us-central1-a # Delete completely (removes everything) gcloud compute instances delete my-first-vm --zone=us-central1-a
